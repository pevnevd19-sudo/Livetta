# Livetta — конструктор, склад, заказы и Excel

## Что изменено

### Конструктор

- Для колье доступны размеры от **30 до 50 см**.
- Размеры браслета остались от 14 до 24 см.
- Добавлен обязательный выбор замка:
  - карабин из нержавеющей стали;
  - тоггл из нержавеющей стали;
  - магнитный замок из нержавеющей стали;
  - винтовой замок из нержавеющей стали;
  - замок-крючок из нержавеющей стали.
- Без выбранного замка сборку нельзя добавить в корзину.
- Тип замка сохраняется в корзине, заказе, админке и Excel.
- Вместимость теперь считается по формуле:
  `длина украшения в мм − длина замка − сумма размеров бусин`.
- Для разных типов замков учитывается разная занимаемая длина.
- Сервер повторно проверяет размер, замок и вместимость при оформлении заказа.

### Склад камней

- В админке есть отдельная вкладка **«Склад камней»**.
- После оформления заказа показатель «Доступно» уменьшается сразу за счёт активного заказа.
- После подтверждения оплаты количество переносится в статистику **«Продано камней»**.
- Внутренний резерв сохранён для защиты от двойной продажи, но теперь он называется «В активных заказах».
- При подтверждении оплаты остатки списываются, а статистика продаж увеличивается.
- Админка автоматически обновляет открытую вкладку заказов или камней каждые 10 секунд, если пользователь не редактирует поле.

### Статус «Оплачен»

- Статус заказа теперь можно вручную изменить на **«Оплачен»**.
- При этом автоматически:
  - меняется статус оплаты;
  - списываются товары и камни;
  - увеличивается статистика продаж;
  - создаётся запись в истории заказа.
- Если нужных остатков уже нет, админка покажет ошибку и не позволит подтвердить оплату.

### Пользователи

- Добавлено удаление пользователей.
- Нельзя удалить пользователя, под которым выполнен вход.
- Нельзя удалить, отключить или понизить последнего активного владельца.
- В системе всегда должен оставаться минимум один активный `owner`.

### Внешний вид админки

- Карточки украшений и камней получили более аккуратное оформление.
- Описание товара вынесено в отдельный оформленный блок.
- Цена, категория, доступный остаток и продажи отображаются отдельными метками.
- Формы редактирования скрываются в раскрывающихся блоках.
- Кнопки оформлены как на основном сайте: розовый фон, белый текст и лавандовая обводка.

## Excel-файл заказов

Добавлена работа с настоящим файлом `.xlsx`.

В админке доступны:

- **Скачать Excel**;
- **Обновить Excel-файл**;
- **Загрузить изменения из Excel**.

В Excel можно менять:

- статус заказа;
- стоимость доставки;
- трек-номер;
- комментарий администратора;
- ссылку на чек;
- отметку об отправке чека.

После редактирования загрузите файл обратно через админку. Изменения покупателей, состава и суммы заказа из Excel намеренно не импортируются.

### Автоматический файл на компьютере

Если сервер запускается на вашем Windows-компьютере, добавьте в `.env`:

```env
ORDERS_EXCEL_PATH=C:/Users/TBG/Desktop/Livetta/livetta-orders.xlsx
```

После создания или изменения заказа этот файл будет обновляться автоматически.

Важно: если файл открыт в Excel, Windows может заблокировать его перезапись. Закройте файл и нажмите **«Обновить Excel-файл»**.

Если сайт размещён на хостинге, сервер не может напрямую записывать файл на домашний компьютер. В этом случае файл хранится на сервере, скачивается через админку и загружается обратно после редактирования. Для постоянной синхронизации с компьютером можно позже подключить OneDrive.

## Установка

После замены проекта обязательно выполните:

```bash
npm install
npm start
```

Добавлена новая зависимость `exceljs`, поэтому одного `npm start` после замены недостаточно.

Затем обновите браузер:

```text
Ctrl + F5
```

## База данных

Удалять `data/database.sqlite` не нужно. При первом запуске сервер сам добавит новые поля статистики продаж.
