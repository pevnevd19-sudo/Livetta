# Livetta — проверка функциональности и исправления

## Как запустить

В папке проекта выполните:

```bash
npm install
npm start
```

Сайт:

```text
http://localhost:3000
```

Админка:

```text
http://localhost:3000/admin.html
```

Логин и пароль берутся из файла `.env`.

Если вы изменили `ADMIN_LOGIN` или `ADMIN_PASSWORD`, выполните:

```bash
npm run reset-owner
npm start
```

Это обновит владельца в `data/database.sqlite`, не удаляя товары, камни и заказы.

## Что исправлено

### Авторизация

- Исправлена причина постоянной ошибки `401 Unauthorized`.
- Пароль владельца в базе синхронизирован с данными из `.env`.
- После успешного входа счётчик неудачных попыток сбрасывается.
- Добавлена понятная подсказка в консоли, если пароль в базе не совпадает с `.env`.
- Скрипт `reset-owner.js` больше не выводит пароль в терминал.
- Последнего активного владельца теперь нельзя случайно отключить или понизить в роли.

### Заказы и оплата

- Исправлено сохранение статуса заказа: изменение доставки больше не сбрасывает статус.
- В админке данные доставки и чека сохраняются до изменения статуса.
- Отметку «Чек отправлен» теперь можно как поставить, так и снять.
- Исправлена финализация склада после подтверждения оплаты ЮKassa.
- При отмене платежа резерв освобождается корректно.
- Добавлены уведомления об ошибках для кнопки создания платёжной ссылки.
- Кнопки удаления требуют повторного нажатия, поэтому заказ нельзя удалить случайно.

### Товары, категории и склад

- Нельзя уменьшить остаток товара или бусины ниже уже зарезервированного количества.
- Нельзя удалить товар или камень, если он участвует в активном резерве.
- Нельзя удалить категорию, пока в ней находятся товары.
- Загрузка изображений теперь сохраняет безопасное расширение по реальному MIME-типу файла.
- Исправлены дублирующиеся элементы в интерфейсе категорий.
- Исправлена обработка ошибок добавления, редактирования и удаления товаров и камней.
- Добавлены уведомления об успешном сохранении и причинах ошибок.

### Карусель и главная страница

- Проверены загрузка, удаление и изменение порядка слайдов.
- Проверено сохранение популярных товаров.
- Добавлены состояния загрузки и понятные сообщения об ошибках.
- Параметры стрелок карусели не изменялись:
  - назад: `left: 18px; padding-bottom: 7px`;
  - вперёд: `right: 18px; padding-bottom: 7px`.

### Пользователи и резервные копии

- Проверены создание и изменение пользователей.
- Новый пароль должен быть не короче 8 символов.
- Защищён последний активный владелец.
- Исправлены сообщения при создании резервной копии.
- Проверена выгрузка заказов в CSV.

### Внешний вид админки

- Админка приведена к стилю основного сайта Livetta.
- Добавлены стеклянные светлые карточки, розовые кнопки и лавандовая обводка.
- Переработана верхняя панель с профилем, экспортом, резервной копией и выходом.
- Формы и поля получили одинаковые размеры и состояния фокуса.
- Изображения товаров и камней остаются квадратными, а не овальными.
- Добавлены цветовые статусы заказов.
- Добавлены всплывающие уведомления об успехе и ошибках.
- Улучшено отображение на телефонах.

## Что нельзя полноценно проверить без ваших ключей

Реальные операции следующих сервисов требуют зарегистрированных аккаунтов и ключей:

- ЮKassa;
- СДЭК;
- Яндекс Почта SMTP;
- Яндекс Метрика.

Код обработки оставлен в существующей архитектуре и подготовлен к подключению через `.env`.
