Livetta bugfix: товары, фото товаров, размеры камней

Что исправлено:
1. В admin.html добавлено поле productImage. Без него admin.js падал при добавлении товара.
2. Форма товара теперь отправляет данные через FormData и может передавать фото товара.
3. server.js хранит image у товаров, size_mm и color у камней.
4. server.js сам добавляет недостающие колонки в старую database.sqlite.
5. server.js принимает размер камня как size_mm, stoneSize, stone_size или size.
6. Цвет камня больше не затирается принудительно.
7. В каталоге есть фото товара и кнопка Купить.
8. В конструкторе размер камня влияет на размер изображения бусины.

Как ставить:
1. Сохрани свои .env и database.sqlite.
2. Замени файлы проекта файлами из архива.
3. Перезапусти сервер: npm start
4. В браузере нажми Ctrl+F5.
5. Для старых камней открой админку и нажми Сохранить у каждого камня, указав размер и цвет.
