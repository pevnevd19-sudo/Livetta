# Livetta — версия для подготовки к полноценному запуску

## Что уже добавлено

- юридические страницы: оферта, политика персональных данных, согласия, оплата, возврат, гарантия, контакты;
- юридический подвал на публичных страницах;
- обязательное согласие с офертой и обработкой данных в корзине;
- отдельное необязательное согласие на рекламную рассылку;
- cookie-баннер и отложенное подключение Яндекс Метрики;
- расширенная форма заказа;
- серверный пересчёт цен товаров и бусин;
- резервирование товара и бусин на время оплаты;
- автоматическое списание остатков после подтверждения оплаты;
- скрытие точных остатков от покупателей;
- склад готовых изделий и бусин в админке;
- роли: владелец, администратор, мастер;
- история заказа, трек-номер, комментарий администратора и ссылка на чек «Мой налог»;
- статусы заказа;
- экспорт заказов в CSV;
- локальное резервное копирование базы;
- интеграционный модуль ЮKassa;
- расчёт курьера Livetta: 700 ₽ в пределах КАД;
- бесплатная доставка по России от 10 000 ₽;
- автоматический расчёт СДЭК после добавления API-ключей;
- страницы результата оплаты, robots.txt, sitemap.xml и 404;
- email-уведомления после настройки SMTP.

## Обязательные действия до приёма реальных денег

1. Вставить ИНН во все документы и подвал вместо текста `781165194751`.
2. Скопировать `.env.example` в `.env` и заполнить секреты.
3. Сменить `ADMIN_PASSWORD` и `JWT_SECRET`.
4. Подать уведомление об обработке персональных данных в Роскомнадзор.
5. Зарегистрироваться в ЮKassa как самозанятая и получить `shopId` и секретный ключ.
6. В личном кабинете ЮKassa указать webhook:
   `https://livettajew.ru/api/payments/yookassa/webhook`
7. Создавать чек НПД в «Моём налоге» и сохранять ссылку на чек в заказе.
8. Получить API-ключи СДЭК, если нужен автоматический расчёт.
9. Получить пароль приложения Яндекс Почты для SMTP.
10. Подключить HTTPS.
11. Проверить, что сервер и первичная база персональных данных находятся в России.
12. Провести тест: заказ → оплата → webhook → списание склада → чек → доставка → возврат.
13. Перед публикацией отдать оферту и политику на финальную проверку профильному юристу.

## Детские украшения

Категория видима, но покупки блокируются настройкой:

`CHILD_PRODUCTS_ENABLED=false`

Это сделано намеренно. Включать `true` следует только после оформления документов о соответствии для детской продукции и проверки маркировки. Без этого карточки могут быть видны как будущий ассортимент, но оплатить их нельзя.

## Установка локально

```bash
npm install
cp .env.example .env
npm start
```

Открыть:

- сайт: `http://localhost:3000`
- админка: `http://localhost:3000/admin.html`

## ЮKassa

Заполнить:

```env
YOOKASSA_SHOP_ID=
YOOKASSA_SECRET_KEY=
```

Пока ключей нет, реальная платёжная ссылка не создаётся. Для локальной проверки можно временно включить:

```env
NODE_ENV=development
PAYMENT_DEMO_MODE=true
```

В production деморежим не работает.

## Доставка

- Санкт-Петербург в пределах КАД: 700 ₽.
- Оплата наличными или СБП при получении доступна только для собственного курьера в пределах КАД.
- СДЭК рассчитывается автоматически при наличии индекса и API-ключей.
- Почта России и другие службы пока требуют подтверждения стоимости в админке.
- После подтверждения доставки администратор создаёт ссылку на оплату и отправляет её покупателю.

## Остатки и роли

- Владелец: весь функционал, пользователи, удаление заказов, резервные копии.
- Администратор: заказы, товары, категории, склад, карусель.
- Мастер: только остатки бусин.

Покупателям точное количество остатков не показывается. Они видят только доступность товара.

## Резервные копии

Ручная копия создаётся в админке или командой:

```bash
npm run backup
```

Файл сохраняется в папку `backups`. Для копирования в облако нужно выбрать конкретное облачное хранилище и настроить его ключи либо синхронизацию. В текущем архиве облачный пароль не хранится.

## Важное ограничение

Архив содержит техническую и документальную основу, но не может сам:

- зарегистрировать ЮKassa;
- подать уведомление в Роскомнадзор;
- получить сертификаты на детские товары;
- определить купленный тариф Majordomo;
- выпустить SSL без доступа к панели;
- гарантировать приём любой иностранной карты;
- заменить индивидуальную юридическую проверку.
